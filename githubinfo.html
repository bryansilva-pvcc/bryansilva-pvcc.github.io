All about Git and GitHub

Questions and Answers

Questions Answers
What is a version control system? A system that records changes to a file or set of files over time, also known as source control or revision contrl.
What is Git? A free and open-source version control system. Used for tracking changes in source code, enabling multiple developers to work collaboratively.
What is GitHub? A developer platform that allows developers to create, store, manage and share their code.
What is the difference between Git and GitHub? GitHub is a hosting service for your Git repositories. GitHub is a service or GUI as to Git is a software used in the command-line.
Who started GitHub and how was it started?? Started by Tom Preston-Werner, Chris Wanstrath, P.J. Hyett and Scott Chacon. Development began on October 19, 2007 and the site was launched in April 2008.
What company owns it now? Owned by Microsoft Corp.
How much does a GitHub account cost? GitHub offers both free and paid plans. Plans starting from $4/month to $20/month..
What is the Octocat? The Octocat is GitHub's mascot created by graphic designer Simon Oxley.

Git/GitHub Terms

Terms Definitions
repsitory a place where you can store your code, files, and each file's revision history.
commit or "revision", is an individual chage to a file. This command captures a snapshot of the projects currently staged changes.
fork is a copy of another's users repository thath lives on your account. This allows you to make changes without affecting the orginal repository.
push means to send your committed changes to a remote respository on your GitHub account.
pull requests are proposed changes to a respositoy submitted by another developer and is either accepted or rejected by the collaborators.
worklflows is a collection of correlated GitHub actions defined in a single file. A configureable automated process made up of one or more jobs.
issues are suggested improvements, tasks or questions related to the respository.
raw button when clicked you can view or copy the raw content of a file without any styling.
blame button when clicked allows you to view a line-by-line revision history, with the code in a file seperated by commit.
