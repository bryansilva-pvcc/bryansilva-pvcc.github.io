Chapter 6: Programming Languages
Terms Definitions
Machine Language represented by binary code, its very difficult for humans to interpret.
Debugging errors from writing programs in machine language that must be located and corrected.
Assembler converts mnemonic expressions into machine language insturctions.
Assembly language a mnemonic system for representing programs.
Tranlator translates programs expressed in high-level primitives into machine-language programs.
Compiler a translation program, that often compiles several machine instructions into short sequences to simulate the activity requested by a single high-level primitive.
Interpreter another means of implementing third-generation Languages, alternative to translators.
Cross-platform software software that is independent of an operating system's design as well as the machine's hardware desing and is therefore executable throughout a network.
Natural language languages such as Enlish, German,and Latin
Formal language are programming languages
Machine independence after the development of third-generation languages, programs were able to be used on any machine.
High-level programming language provide more abstractions for the programmer and make it easier for people to read and write a program.(ie: Python, JavaScript, C, ...)
Object-Oriented Programming(OOP) a programming process associated with programming paradigm based on the concept of objects that interact with one another.
Scripting language a subset of the imperative programming languages is the collection of languages.
Data types
    integer: numeric datat consisting of whole numbers, stored using two's complement notation.
    float: numeric data that might contain values other than whole numbers, stored in floating-point notation.
    string: used to represent text rather than numbers and are enclosed in quotations marks.
Primitive data type programming language such as "int" for integer and "char" for character.
Data structure the conceptual shape or arrangement of data.
Python collection data structures
    list: used to store multiple items in a single variable, created using square brackets.
    tuple: used to store multiple items in a single variable, created using round brackets, a tuple is a collection which is ordered and unchangeable.
    set: used to store multiple items in a single variable, created using curly brackets, a set is a collection which is unordered, unchageable, and unindexed.
    dictionary: used to store data values in key: value pairs, written in curly brackets, and have keys and values. This collection is ordered, changeable and do not allow duplicates.
Function a set of insturctions for performing a task that can be used as an abstract tool by other program units.
Local variable a variable declared within a function
Global variable variables whose scopes are not restricted to a particular part of a program.
Source program a program in its orginal form.
Object program a translatd version of a program.
