Chapter 7: Software Engineering
Terms Definitions
Software Engineering is the branch of computer science that seeks principles to guide the development of large, complex software systems.
Integrated Development Environment(IDE) tools for developing software (editors, complilers, debugging tools) EX: Visual Studio Code, Atom, Xcode.

Software Life Cycle:
Development Phase
    Requirements analysis:
    specify what services the system will provide, also identify any conditions (time constraints, security, etc.)
    Design:
    creating a plan for the construction of the propsed system.
    Implementation:
    involves the actual writing of programs, creation of data files, and development of databases.
    Testing:
    the process of debugging programs and confirming that the final software product was compatible with the software requirments specification
Modular Implementation Modules come in a varity of forms. In context of imperative paradigm, modules appear as functions. In constrast, the object-oriented paradigm uses objects as the basic modular constituents.
Unified Modeling Language (UML) a modern collection of tools for representing the internal object oriented design of a system.
use case depicted in a diagram as a large retangle which interactions between a system and its users.
use case diagram a tool used to dipic an image of the proposed system from the user's point of view.
Data Relationships:
    one-to-one:
    entities in which only having one relationship to another entity. Example in textbook: each patient associated with only one room
    one-to-many:
    one entitie in relationship to multiple enties. Example in textbook: one physician is associated with many patients
    many-to-many:
    entities share relationships with mutiple entites. Example in textbook: each physician could be associated with many patients and each patient could be associated with many physicans.
Design Pattern a predeveloped model for solving a recurring porblem in software design.
Software Quality Assurance (SQA) groups charged with overseeing and enforcing the quality control systems adopted by the orginization.
Software Testing:
    glass box testing:
    a software tester being aware of the interior structure, uses this knowledge when disgning the test.
    black box testing:
    tests that do not rely on the knowledge of the software's interior composition.
    beta testing:
    falls within the black box catagory, a preliminary version of the software given to intended audience with the goal of learning how the software perform in real-life situation before finalizing final version.
    alpha testing:
    similar to beta testing, performed at developers site.
Documentation
    user documentation:
    intended to be read by user, explains the features of teh software and describes how to use them.
    system documentation:
    describes the software's internal composition so that it can be maintained later. Is is the source version of all the programs in the system.
    technical documentation:
    describes how a software system should be installed and serviced.
Intellectual Property Law
    software license:
    a legal agreement between the owener and user of a software product. Grants user permissions without transferring ownership rights.
    software disclaimer:
    to addresses the issue of liability, included in the software licenses states the limitations of their liability.
    best defenses against software liability claims:
    is to apply sound software engineering principles during development, use a level of care complatible with the application, and to produce and maintain records.
